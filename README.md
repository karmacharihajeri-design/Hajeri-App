# रोजची हजेरी अॅप — वेब आवृत्ती (HTML + JS + Google Apps Script)

ही आवृत्ती पूर्णपणे **स्टॅटिक वेब अॅप** आहे — कोणताही सर्व्हर/होस्टिंग खर्च न करता
**GitHub Pages** वरून मोफत चालवता येते. बॅकएंड म्हणून Google Apps Script (Sheets + Drive) वापरला आहे.

---

## नवीनतम अपडेट (सारांश)

- **चेहरा-कॅप्चर बग दुरुस्त**: `face-api.js` स्क्रिप्ट आधी `defer` ठेवली होती, त्यामुळे ती इतर स्क्रिप्ट्सनंतर लोड व्हायची आणि कॅमेरा उघडूनही चेहरा कधीच "ओळखला" जायचा नाही. आता `defer` काढून योग्य क्रमाने लोड होईल असं केलं आहे.
- **सहाय्यक चेहरा-नोंदणी व सहाय्यक हजेरी**: Admin/Developer पॅनलमध्ये "कर्मचाऱ्यासाठी सहाय्यक सुविधा" विभागातून, कामगाराचा मोबाईल नंबर टाकून, Admin/Developer स्वतःच्या डिव्हाइसवरून त्या कामगाराची चेहरा-नोंदणी किंवा हजेरी घेऊ शकतात (कामगार समोर उभा राहून, स्वतःच्या मोबाईलमध्ये अडचण असल्यास).
- **Admin ला फक्त "User" रोल तयार करता येतो**: Admin पॅनलच्या "नवीन स्टाफ जोडा" फॉर्ममध्ये आता "Admin" हा पर्यायच दिसत नाही — फक्त Developer ला नवीन Admin तयार करता येतो (सर्व्हरवरही ही तपासणी आहे, फक्त पानावर लपवलेली नाही).
- **डिव्हाइस-लॉक फक्त कामगारांसाठी**: Admin आणि Developer आता **कुठल्याही डिव्हाइस/ब्राउझरवरून** लॉगिन करू शकतात — डिव्हाइस-लॉक फक्त `user` (कामगार) रोलला लागू होतो.
- **Logout + Exit दोन्ही बटणे, रंगीत बॉर्डरसह**: प्रत्येक पानावर आता स्पष्ट "लॉगआउट" व "बाहेर पडा" बटणे आहेत, आणि सर्व बटणांना ठळक रंगीत बॉर्डर दिली आहे.
- **Logout/Exit नंतर Back बटण काम करत नाही**: नवीन `auth-guard.js` मुळे लॉगआउट/एक्झिट केल्यावर ब्राउझरचं Back बटण दाबलं तरी आतल्या (संरक्षित) स्क्रीनवर परत जाता येत नाही — थेट लॉगिन स्क्रीनच दिसते.
- **Admin ला आता PDF हजेरी पत्रक तयार करता येते** ("परवानगी नाही" ही जुनी त्रुटी दुरुस्त झाली).
- **PDF डिव्हाइसवर सेव्ह + प्रिंट**: PDF तयार झाल्यावर "डिव्हाइसवर सेव्ह करा" बटणाने फोल्डर निवडून सेव्ह करता येते (Chrome/Edge वर थेट फोल्डर-निवड डायलॉग; इतर ब्राउझरवर सामान्य डाउनलोड), आणि "प्रिंट करा" बटणाने नवीन टॅबमध्ये उघडून थेट प्रिंट डायलॉग (वायरलेस/कोणताही जोडलेला प्रिंटर निवडण्याचा पर्याय त्यातूनच मिळतो). हे सर्व Drive वरील सेव्हच्या **व्यतिरिक्त** आहे — दोन्ही एकत्र होतात.
- **PDF मध्ये अर्धी हजेरी/रजा दिसत नव्हती — मूळ बग सापडून दुरुस्त**: Google Sheets कधीकधी "yyyy-MM-dd" स्वरूपातल्या तारखेच्या मजकुराला आपोआप Date ऑब्जेक्टमध्ये बदलतं — त्यामुळे जुना कोड सरळ (`===`) तुलना करत असल्याने ती जुळतच नव्हती, आणि प्रत्यक्ष हजेरी नोंदवलेली असूनही अहवालात नेहमी "गैरहजर" दिसायचा. आता सर्व तारीख-तुलना `normalizeDateStr_()` मधून जातात — त्यामुळे पूर्ण/अर्धी हजेरी व रजा दोन्ही अहवालात व आजच्या स्थितीत बरोबर दिसतात.
- **कामानिमित्त बाहेर (Outdoor Duty) नोंद**: कर्मचारी नेमून दिलेल्या ठिकाणाबाहेर असल्यास हजेरीच्या वेळी "📍 आज कामानिमित्त नेमून दिलेल्या ठिकाणाबाहेर आहे" हा पर्याय निवडून कारण + सध्याचे ठिकाण लिहू शकतो — अशा वेळी GPS जिओफेन्सची अट वगळली जाते (चेहरा-ओळख व फोटो मात्र आवश्यक असतातच), आणि ती नोंद Attendance शीट व PDF अहवालात कारणासह दिसते.
- **Standby (निष्क्रियता) वेळ**: Developer आता Admin पॅनलमधून ठरवू शकतो की किती मिनिटे काहीच हालचाल न झाल्यास अॅप आपोआप लॉगआउट होईल (सार्वजनिक/शेअर्ड डिव्हाइससाठी सुरक्षितता — 0 = बंद).
- **सुट्ट्यांची यादी आता शोधण्यायोग्य dropdown**: Developer साठी संपूर्ण यादी दाखवण्याऐवजी टाइप-करून-शोधणारा बॉक्स (datalist) दिला आहे — सुट्ट्यांची संख्या वाढली तरी यादी अवजड होणार नाही.
- **नकाशावरील रस्ता/सॅटेलाइट बटणे आता मोबाईलवर नकाशा झाकत नाहीत**: आधीचं Leaflet चं built-in टॉगल मोबाईलवर मोठी जागा घेत असल्याने जागा निवडणं अवघड व्हायचं — आता ते नकाशाच्या *खाली* दोन छोट्या रंगीत बटणांच्या स्वरूपात दिले आहे. तसंच नकाशा उघडताच वापरकर्त्याच्या सध्याच्या GPS स्थानावर आपोआप केंद्रित होतो (आधीपेक्षा जास्त विश्वासार्ह टाइमआउट/अचूकतेसह).
- **"user" (साधा कामगार) रोलला आता Admin लिंक दिसत नाही** — हजेरी पानावरील "Admin →" लिंक फक्त Admin/Developer लॉगिनलाच दिसते.
- **Admin पॅनलमध्ये "हजेरी पत्रक तयार करा" आता सर्वात वर, ठळक** — मोठा, केंद्रित, गडद-मरून जाड बॉर्डरचा बटण थेट पानाच्या वरच दिला आहे.
- **लॉगिन स्क्रीनवरील Exit बटण अधिक सक्षम** — शक्य तितक्या प्रयत्नाने टॅब बंद करतं; ब्राउझर सुरक्षिततेमुळे (कुठल्याही वेबसाईटला) पूर्णपणे बंद करता आलं नाही, तर संपूर्ण पान साफ करून "✅ अॅप बंद झाला" असा स्पष्ट पूर्ण-स्क्रीन संदेश दाखवतं.
- **फोटो आता पूर्णपणे आपोआप — बटण दाबायची गरज नाही**: चेहरा नोंदणी (enroll.html) मध्ये आता प्रत्येक पोझ (सरळ/डावे/उजवे/वर/खाली) चेहरा स्थिर दिसताच आपोआप कॅप्चर होते — "हा कोन कॅप्चर करा" बटण दाबायची गरज उरली नाही, प्रत्येक कॅप्चरनंतर वरती छोटी सूचना (toast) दिसते. हजेरीच्या वेळीही (attendance.html) चेहरा जुळून, वेळ बरोबर असून, GPS मिळाल्यावर हजेरी आपोआप नोंदवली जाते — बटणे फक्त बॅकअप/मॅन्युअल पर्याय म्हणून तशीच राहतात.
- **चेहरा आता जास्त लवकर व विश्वासार्हपणे सापडतो**: आधी एखादी फ्रेम चुकली (किंचित हालचाल, प्रकाश) तरी प्रगती पूर्ण शून्यावर जायची, त्यामुळे कॅमेऱ्यासमोर बराच वेळ राहूनही कॅप्चर व्हायचं नाही. आता चेहरा-शोध जलद (छोटा inputSize, कमी scoreThreshold) असून चुकलेल्या फ्रेमने प्रगती फक्त १ ने कमी होते (पूर्ण रीसेट होत नाही) — त्यामुळे कॅप्चर लवकर व सहज होतं.
- **मोठा बग सापडून दुरुस्त: हजेरीच्या वेळा चुकीच्या दिसणे व चेहरा जुळूनही हजेरी न नोंदवली जाणे**: Google Sheets कधी-कधी "09:00" सारखा वेळेचा मजकूर आपोआप Date/Time व्हॅल्यूमध्ये बदलतं — त्यामुळे सकाळ/दुपार हजेरीच्या वेळा पडद्यावर चुकीच्या दिसायच्या, आणि सर्व्हरवरील वेळ-तुलना (isWithinWindow_) कायम अवैध (NaN) ठरून हजेरी नाकारली जायची — जरी चेहरा व्यवस्थित जुळला व वेळही बरोबर असली तरी. आता ही वेळ नेहमी योग्य "HH:mm" मजकुरात रूपांतरित करून वाचली जाते, व Settings शीटचा Value कॉलम कायमचा "Plain Text" फॉरमॅटमध्ये ठेवला आहे जेणेकरून ही समस्या परत उद्भवणार नाही.
- **चेहरा जुळल्यावर हजेरी आता जवळपास लगेच नोंदवली जाते**: आधी हजेरीसाठी सलग साधारण दीड सेकंद स्थिर चेहरा हवा असायचा; आता जुळल्यावर लगेच (सुमारे अर्धा सेकंदात) फोटो घेतला जातो — नंतर सर्व्हरला पाठवताना चेहरा कॅमेऱ्यासमोर राहण्याची गरज नाही (फोटो आधीच त्या क्षणी घेतलेला असतो).
- **कॅमेऱ्यासमोर वाट पाहताना आता मध्येच लॉगआउट होत नाही**: Standby (निष्क्रियता) टायमर आधी फक्त टच/क्लिकलाच "activity" धरत होता — चेहरा नोंदणी/हजेरीच्या वेळी नुसतं कॅमेऱ्यासमोर स्थिर उभं राहणं ही हालचाल गणली जात नव्हती, त्यामुळे स्टँडबाय वेळ सेट केलेली असल्यास मध्येच लॉगआउट व्हायचं. आता कॅमेरा सक्रिय असताना चेहरा-शोध सुरू असणं हीदेखील "activity" धरली जाते.
- **हजेरीची वेळ आता ठळक बॅज स्वरूपात दिसते**: प्रत्येक बटणाखाली त्या सत्राची वेळ (उदा. "🌅 सकाळ: ९:०० ते ११:००") आता एका रंगीत, ठळक बॅजमध्ये दिसते.
- **तुम्ही नेमून दिलेल्या ठिकाणापासून किती लांब आहात हे आता स्क्रीनवर लाईव्ह दिसतं**: GPS मिळताच "✅ तुम्ही [ठिकाण] या ठिकाणी आहात" किंवा "⚠️ तुम्ही [ठिकाण] पासून X मी. लांब आहात — जवळ जा" असा संदेश दिसतो, जेणेकरून हजेरी नोंदवण्याआधीच कर्मचाऱ्याला कळेल की आणखी जवळ जायचं आहे का.
- **आतल्या पानांवरील "बाहेर पडा (Exit)" बटण काढून त्याऐवजी सर्वात वर "← मागे जा" बटण**: फक्त लॉगिन स्क्रीनवरचं Exit बटण कार्यरत आहे (व ते आता जास्त सक्षमपणे टॅब/अॅप बंद करण्याचा प्रयत्न करतं, न जमल्यास स्पष्ट "अॅप बंद झाला" असा पूर्ण-स्क्रीन संदेश दाखवतं). इतर सर्व आतल्या पानांवर (हजेरी, Admin, रजा, अहवाल, स्पॉट-असाइन, चेहरा-नोंदणी) आता सर्वात वर एक "← मागे जा" बटण आहे.
- **संपूर्ण अॅपचं रंगसंगती अपडेट** — सर्व बटणे व nav-लिंक्स आता जास्त रंगीत, ठळक फॉन्ट व हलक्या hover-इफेक्टसह, अधिक व्यावसायिक दिसण्यासाठी.
- **सकाळ/दुपार दोन-सत्र हजेरी पद्धत (मोठा बदल)**: आता एकाच जनरल Check-In/Check-Out ऐवजी सकाळ आणि दुपार अशा दोन स्वतंत्र हजेरी-सत्रांची पद्धत आहे —
  - Developer, Admin पॅनलमध्ये सकाळच्या व दुपारच्या हजेरीची वेळ (From-To) ठरवतो — उदा. सकाळ ९:०० ते ११:००, दुपार ३:३० ते ७:००.
  - त्या-त्या वेळेतच संबंधित हजेरीचा फोटो काढता येतो — इतर वेळी बटण आपोआप लॉक (निष्क्रिय) राहते.
  - फक्त सकाळचा फोटो → "सकाळची अर्धी हजेरी" | फक्त दुपारचा फोटो → "दुपारची अर्धी हजेरी" | दोन्ही → "पूर्ण हजेरी" | दोन्हीपैकी एकही नाही → "गैरहजर".
  - एकदा एखाद्या सत्राचा फोटो नोंदवला की तोच सत्र पुन्हा नोंदवता येत नाही (एका कामगाराची एका सत्राची हजेरी एकदाच).
  - `attendance.html` वर आता दोन स्वतंत्र बटणे आहेत — "🌅 सकाळची हजेरी" व "🌇 दुपारची हजेरी" — प्रत्येक फक्त त्याच्या ठरलेल्या वेळेत, चेहरा जुळल्यावर व GPS मिळाल्यावरच सक्रिय होते.
- **सुट्ट्या आपोआप ओळखणे**: शनिवार-रविवार आपोआप साप्ताहिक सुट्टी धरली जाते (Settings मधील `WeeklyOffDays` वरून बदलताही येते). याशिवाय Developer Admin पॅनलमधून सण/शासकीय सुट्या ("Holidays" शीट) जोडू शकतो. सुट्टीच्या दिवशी फोटो काढण्याची गरज नसते, हजेरी बटणे आपोआप लपतात, आणि PDF अहवालात त्या तारखेसमोर आपोआप "सुट्टी" अशी नोंद (वेगळ्या रंगात) येते.
- **रजा अर्ज प्रणाली (नवीन)**: कामगार कुठूनही लॉगिन करून नवीन `leave.html` पानावरून रजेचा अर्ज सादर करू शकतो — रजेच्या अर्जाचा फोटो/PDF पुरावा अपलोड करून From-To तारखा भरणे. सादर झाल्यावर त्या तारखांना हजेरीची गरज राहत नाही (हजेरी पानावर आपोआप "रजा" असा संदेश दिसतो, कॅमेरा/GPS सुरूच होत नाही) व PDF अहवालातही "रजा" अशी नोंद येते. कामगार रजेच्या तारखेआधी कामावर हजर झाल्यास, Admin पॅनलमधील "कर्मचाऱ्यांचे रजा अर्ज" विभागातून Admin किंवा Developer त्या रजेच्या तारखा (उदा. शेवटची तारीख कमी करून) बदलू शकतात.
- **PDF अहवालात सुट्टी/रजा/अर्धी-पूर्ण हजेरी सर्व एकत्र**: प्रत्येक दिवसासाठी योग्य ती नोंद (पूर्ण/अर्धी हजेरी, गैरहजर, सुट्टी, रजा) आपोआप ठरते व शेवटी हजर/गैरहजर/सुट्टी/रजा दिवसांची एकूण बेरीजही दाखवली जाते.
- **चेहरा-कॅप्चर बग पुन्हा दुरुस्त (मूळ कारण सापडले)**: कॅमेरा उघडायचा पण "हा कोन कॅप्चर करा" बटण कधीच सक्रिय व्हायचं नाही — कारण कॅमेरा सुरू झाल्या-झाल्या लगेच चेहरा शोधणं सुरू व्हायचं, पण व्हिडिओला अजून प्रत्यक्ष फ्रेम्स मिळालेल्याच नसायच्या (काही मोबाईलवर `autoplay` असूनही व्हिडिओ आपोआप चालू होत नाही). त्यामुळे कायम रिकामी (0x0) फ्रेम तपासली जायची. आता `face-utils.js` मधील `startCamera()` व्हिडिओ खरंच play होईपर्यंत थांबते, आणि `detectFaceDescriptor()` आधी व्हिडिओ तयार आहे का हे तपासते.
- **PDF "डिव्हाइसवर सेव्ह करा" — iOS साठी सुधारित**: iPhone/iPad वर आधीची डाउनलोड पद्धत बरेचदा शांतपणे अयशस्वी व्हायची. आता iOS ओळखून PDF नवीन टॅबमध्ये उघडते व "Share (□↑) → Save to Files" वापरायची स्पष्ट सूचना दाखवते. Android/Desktop वर आधीचीच पद्धत (फोल्डर-निवड डायलॉग / सामान्य डाउनलोड) चालू आहे.
- **PDF "प्रिंट करा" — जास्त विश्वासार्ह पद्धत**: आधी नवीन टॅब उघडून लगेच `print()` कॉल व्हायचा, पण PDF व्ह्यूअर तोपर्यंत लोडच झालेला नसायचा, त्यामुळे प्रिंट डायलॉग उघडायचाच नाही. आता लपवलेल्या iframe मध्ये PDF पूर्ण लोड झाल्यावरच प्रिंट डायलॉग उघडला जातो (Chrome/Edge/Firefox/Android वर विश्वासार्ह; iOS वर नवीन टॅब उघडून Share मेनूतील Print वापरायला सांगितलं जातं).
- **नकाशावर सॅटेलाइट व्ह्यू**: `spot-assign.html` च्या "🗺️ नकाशावरून निवडा" टॅबमध्ये आता उजव्या-वरच्या कोपऱ्यात "🗺️ रस्ता नकाशा" / "🛰️ सॅटेलाइट" असा टॉगल आहे (Esri चे मोफत सॅटेलाइट टाइल्स — कुठलीही API-key लागत नाही).
- **लॉगिन स्क्रीनवर Exit बटण**: आता लॉगिन कार्डाखाली थेट "बाहेर पडा (Exit)" बटण आहे — लॉगिन करण्याआधीही अॅपमधून बाहेर पडता येते.

### १. बॅकएंड (Google Sheet + Apps Script) — आता पूर्णपणे आपोआप

## हजेरीचा प्रवाह — सकाळ/दुपार दोन सत्रे

1. Developer Admin पॅनलमधून सकाळच्या व दुपारच्या हजेरीच्या वेळा (From-To) ठरवतो — उदा. सकाळ ९:०० ते ११:००, दुपार ३:३० ते ७:००.
2. कर्मचारी `attendance.html` उघडतो — जर आज साप्ताहिक सुट्टी/शासकीय सुट्टी/मंजूर रजा असेल, तर कॅमेरा/GPS सुरूच होत नाही, फक्त तसा संदेश दिसतो.
3. नाहीतर कॅमेरा व GPS सुरू होतात — "🌅 सकाळची हजेरी" व "🌇 दुपारची हजेरी" अशी दोन स्वतंत्र बटणे दिसतात.
4. प्रत्येक बटण **फक्त** त्याच्या ठरलेल्या वेळेत, चेहरा नोंदवलेल्या कर्मचाऱ्याशी जुळल्यावर, व GPS कामाच्या ठिकाणाच्या मर्यादेत असेल तरच सक्रिय होते. वेळ संपली/अजून सुरू नाही तर बटण आपोआप निष्क्रिय राहते (दर १५ सेकंदांनी आपोआप तपासले जाते).
5. एकदा एखाद्या सत्राचा फोटो नोंदवला की तेच बटण "✅ ... हजेरी झाली" दाखवून कायमचं निष्क्रिय होते — तोच सत्र पुन्हा नोंदवता येत नाही.
6. स्थिती: फक्त सकाळ = "सकाळची अर्धी हजेरी" | फक्त दुपार = "दुपारची अर्धी हजेरी" | दोन्ही = "पूर्ण हजेरी" | दोन्हीपैकी काहीच नाही (व सुट्टी/रजा नसेल तर) = "गैरहजर".

## चेहरा नोंदणी

1. Admin नवीन कर्मचारी जोडतो (मोबाईल नंबर + ठिकाण).
2. कर्मचारी पहिल्यांदा लॉगिन करतो → त्याचा चेहरा नोंदवलेला नसल्याने आपोआप **enroll.html** वर पाठवला जातो.
3. **मल्टी-अँगल नोंदणी**: कर्मचारी ५ वेगवेगळ्या पोझमध्ये चेहरा कॅप्चर करतो — सरळ समोर, डावीकडे, उजवीकडे, वर, खाली. हे ५ गणितीय "चेहरा ठसे" (128-अंकी अ‍ॅरे, फोटो नाही) Sheet मध्ये एकत्र साठवले जातात.
4. पुढच्या प्रत्येक हजेरीच्या वेळी **attendance.html** लाईव्ह कॅमेरातला चेहरा या ५ ठशांपैकी *कोणत्याही एकाशी* जुळतो का हे तपासतो (`isSameFaceMulti`). यामुळे प्रकाश, कोन, चष्मा, हलकी दाढी यांमुळे खरा कर्मचारीच नाकारला जाण्याची शक्यता कमी होते.
5. गॅलरीतून फोटो निवडण्याचा पर्यायच दिलेला नाही — फक्त लाईव्ह `getUserMedia()` कॅमेरा स्ट्रीम वापरला आहे.
6. **दाढी खूप वाढली / नवीन चष्मा / चेहरा सतत जुळत नसेल तर**: Admin पॅनलमधून ("चेहरा-नोंदणी रीसेट") त्या कर्मचाऱ्याची जुनी नोंदणी रद्द करू शकतो — पुढच्या लॉगिनला तो पुन्हा ५ कोनांतून नोंदणी करेल.

## सुट्ट्या (साप्ताहिक + शासकीय/सार्वजनिक)

- **साप्ताहिक सुट्टी**: शनिवार-रविवार आपोआप सुट्टी धरली जाते (Settings मधील `WeeklyOffDays` की, डीफॉल्ट `0,6` — 0=रविवार, 6=शनिवार — बदलूनही देता येते).
- **शासकीय/सार्वजनिक सुट्या**: Developer Admin पॅनलमधील "सुट्ट्यांची यादी" विभागातून तारीख + नाव टाकून जोडतो (उदा. दिवाळी, स्वातंत्र्य दिन). यादीतून हवी ती सुट्टी "काढा" बटणाने काढताही येते.
- या दोन्ही प्रकारच्या सुट्टीच्या दिवशी कर्मचाऱ्याला फोटो काढण्याची गरज नसते (हजेरी बटणेच दिसत नाहीत), आणि हजेरी पत्रक PDF मध्ये त्या तारखेसमोर आपोआप "सुट्टी" अशी नोंद (वेगळ्या रंगाच्या रांगेत) येते.

## रजा अर्ज प्रणाली

1. कामगार कुठूनही (कामाच्या ठिकाणी नसतानाही) लॉगिन करून `leave.html` उघडतो.
2. रजेच्या अर्जाचा स्पष्ट फोटो किंवा PDF अपलोड करतो, व रजा कोणत्या तारखेपासून कोणत्या तारखेपर्यंत आहे ते भरतो.
3. सादर केल्यावर त्या तारखांसाठी हजेरीची गरज राहत नाही — त्या दिवशी अॅप उघडल्यास आपोआप "रजा मंजूर आहे" असा संदेश दिसतो, कॅमेरा/GPS सुरूच होत नाही.
4. हजेरी पत्रक PDF मध्येही त्या तारखांसमोर आपोआप "रजा" अशी नोंद येते.
5. **कामगार रजेच्या तारखेआधी कामावर हजर झाल्यास**: Admin पॅनलमधील "कर्मचाऱ्यांचे रजा अर्ज" विभागात Admin किंवा Developer त्या रजेच्या From/To तारखा बदलू शकतात (उदा. शेवटची तारीख कमी करून) — तिथूनच रजेच्या अर्जाचा अपलोड केलेला पुरावाही पाहता येतो.
6. एका कामगाराने सादर केलेल्या सर्व जुन्या रजा त्याला स्वतःला `leave.html` वर खालीच दिसतात.

## सेटिंग्ज ऑटो-सिंक (कुठल्याही मोबाईलवर मॅन्युअल सेटअपशिवाय)

- Developer ने Admin पॅनलमधून एकदा अॅपचे नाव/पत्ता सेव्ह केले, की ते Google Sheet च्या `Settings` टॅबमध्ये साठवले जाते.
- **कोणत्याही मोबाईल/डिव्हाईसवर** अॅप (वेबसाईट) उघडताच `settings-sync.js` आपोआप हे सेटिंग्ज आणून पानावर दाखवते — त्या डिव्हाईसवर वेगळे काहीही सेट करावे लागत नाही.
- **लॉगिन मात्र फक्त Developer/Admin ने `Users` शीटमध्ये आधीच नोंदवलेल्या मोबाईल नंबरवरूनच शक्य होते** — इतर कोणत्याही नंबरने लॉगिनचा प्रयत्न केल्यास "नोंदणीकृत नाही" असा संदेश येतो (`loginUser()` फंक्शन पहा). नवीन नंबर वापरायचा असल्यास तो आधी Admin पॅनलमधून जोडावा लागतो.

## कामाचे ठिकाण नोंदवणे (Spot Assign) — दोन मार्ग, फक्त Developer साठी

आधी Admin ला LocationID हाताने टाईप करावा लागत असे. आता `spot-assign.html` वर **फक्त Developer** ला
दोन टॅब दिसतात:

1. **📷 GPS + फोटो** — प्रत्यक्ष जागेवर जाऊन फोटो काढून तेच GPS केंद्रबिंदू म्हणून नोंदवणे.
2. **🗺️ नकाशावरून निवडा** — मोफत OpenStreetMap नकाशावर (Leaflet लायब्ररी, कुठलाही API-key लागत नाही)
   टॅप/क्लिक करून केंद्रबिंदू निवडणे — प्रत्यक्ष जागेवर जाण्याची गरज नाही, पण फोटो-पुरावा राहत नाही.
   नकाशाच्या उजव्या-वरच्या कोपऱ्यातील टॉगलने "🗺️ रस्ता नकाशा" किंवा "🛰️ सॅटेलाइट" व्ह्यू निवडता येतो.

दोन्ही पद्धतीत जागेचे नाव व अनुज्ञेय त्रिज्या (मीटर) टाकून सेव्ह केल्यावर ती जागा `WorkLocations`
शीटमध्ये नोंदवली जाते, आणि लगेच **Admin ला स्टाफ जोडताना ड्रॉपडाऊन यादीत** दिसू लागते —
हाताने LocationID टाईप करावा लागत नाही.

**सुरक्षा**: ही सुविधा फक्त पानावर लपवलेली नाही — बॅकएंडवरही (`requireRole_` तपासणी) फक्त Developer
रोल असलेल्या मोबाईल नंबरलाच नवीन जागा नोंदवता येते; Admin किंवा User ने थेट API कॉल करण्याचा प्रयत्न
केला तरी तो नाकारला जातो.

यूजरला याच नोंदवलेल्या केंद्रबिंदूपासून ठरवलेल्या त्रिज्येतच फोटो काढून हजेरी लावता येते —
बाहेर गेल्यास "मर्यादेबाहेर आहात" असा संदेश येतो.

## डिव्हाइस-लॉक लॉगिन (एक नंबर = एकच डिव्हाइस — फक्त कामगारांसाठी) + नाव-लुकअप

- लॉगिन स्क्रीनवर मोबाईल नंबर टाकताच (१० अंक पूर्ण होताच) त्या नंबरचं नाव आपोआप खाली दिसतं — यूजरला खात्री होते की बरोबर नंबर टाकला आहे.
- **डिव्हाइस-लॉक फक्त `user` (कामगार) रोलला लागू आहे.** Admin आणि Developer कुठल्याही मोबाईल/ब्राउझरवरून केव्हाही लॉगिन करू शकतात — त्यांच्यासाठी कुठलाही डिव्हाइस-निर्बंध नाही.
- प्रत्येक कामगाराचा मोबाईल नंबर **पहिल्यांदा** ज्या डिव्हाइस/ब्राउझरवरून लॉगिन होतो, तोच आपोआप "अधिकृत डिव्हाइस" म्हणून नोंदवला जातो (`Users` शीटच्या `RegisteredDeviceID` कॉलममध्ये, `device.js` द्वारे तयार होणाऱ्या ओळख क्रमांकावरून).
- त्यानंतर **त्याच नंबरने दुसऱ्या मोबाईल/ब्राउझरवरून** लॉगिन करण्याचा प्रयत्न केल्यास "हा मोबाईल नंबर आधीच दुसऱ्या डिव्हाइसवर नोंदणीकृत आहे" असा संदेश येऊन लॉगिन थांबवलं जातं.
- कामगाराने खरंच नवीन फोन घेतला किंवा जुना हरवला, तर Admin पॅनलमधून **"डिव्हाइस-नोंदणी रीसेट करा"** वापरून जुनी नोंदणी रद्द करता येते — पुढच्या लॉगिनला जे नवीन डिव्हाइस वापरेल तेच लॉक होईल.
- **मर्यादा (प्रामाणिकपणे सांगायचं तर)**: ही ओळख ब्राउझरच्या `localStorage` मध्ये साठवली जाते — हार्डवेअर-लेव्हल लॉक नाही. यूजरने ब्राउझरचा डेटा/कॅशे साफ केला किंवा वेगळा ब्राउझर/इनकॉग्निटो मोड वापरला, तर ते नवीन डिव्हाइस म्हणून ओळखलं जाईल. सर्वसामान्य वापरासाठी (एकच फोन, एकच ब्राउझर, कॅशे साफ न करता) हे व्यवस्थित काम करतं.

## कॅमेरा / GPS सुरू होत नसेल तर

आता कॅमेरा व GPS दोन्हीसाठी नेमकं कारण सांगणारे संदेश व **"पुन्हा प्रयत्न करा"** बटण जोडलं आहे (`attendance.html`, `enroll.html`, `spot-assign.html`). सर्वसाधारण कारणं व उपाय:
- **परवानगी नाकारली गेली**: ब्राउझरच्या ॲड्रेस बारमधील 🔒/ⓘ चिन्हावर टॅप करून Camera/Location साठी "Allow" करा, मग पान रिफ्रेश करा (एकदा नाकारल्यावर JS मधून पुन्हा विचारता येत नाही — ब्राउझर सेटिंगमधूनच बदलावं लागतं).
- **HTTP ऐवजी HTTPS हवे**: कॅमेरा/GPS सुरक्षित (https://) पत्त्यावरच काम करतात — `https://karmacharihajeri-design.github.io/...` अशीच लिंक वापरा (GitHub Pages आपोआप HTTPS देते, त्यामुळे साधारणपणे ही समस्या येणार नाही).
- **GPS सिग्नल मिळत नाही**: मोबाईलचं Location/GPS सुरू आहे का ते तपासा, मोकळ्या जागेत/खिडकीजवळ जा.
- **कॅमेरा दुसऱ्या अॅपमध्ये आधीच वापरात आहे**: तो अॅप बंद करून पुन्हा प्रयत्न करा.
- **ब्राउझर जुना/सपोर्ट नसलेला**: Chrome किंवा Safari चं नवीनतम व्हर्जन वापरा.
- **कॅमेरा उघडतो पण चेहरा कधीच "ओळखला" जात नाही / कॅप्चर बटण सक्रिय होत नाही**: हे आता दुरुस्त झाले आहे — व्हिडिओ प्रत्यक्ष सुरू (play) होईपर्यंत चेहरा-शोध थांबतो. तरीही समस्या राहिल्यास, इंटरनेट स्लो असल्याने `FACE_MODEL_URL` (face-api मॉडेल फाईल्स) लोड व्हायला उशीर होत असू शकतो — काही सेकंद थांबून पहा.

## ⚠️ आधार Face RD / e-KYC बद्दल महत्त्वाची सूचना

तुम्ही विचारलेली **आधार लिंकिंग / आधार Face RD e-KYC** सुविधा जोडणे हे **फक्त कोडिंगचा प्रश्न नाही** —
हे भारत सरकारच्या **UIDAI** (Unique Identification Authority of India) कडून **कायदेशीर परवानगी व नोंदणी**
मागणारी बाब आहे:

- आधार-आधारित e-KYC/Face-authentication वापरण्यासाठी तुमची संस्था **AUA/KUA (Authentication User Agency /
  KYC User Agency)** म्हणून UIDAI कडे अधिकृतपणे नोंदणीकृत असावी लागते, किंवा एखाद्या नोंदणीकृत
  Sub-AUA च्या मार्फत काम करावे लागते.
- यासाठी सुरक्षा ऑडिट (STQC प्रमाणपत्र), परवाना शुल्क, आणि Aadhaar Data Vault सारख्या डेटा-सुरक्षा
  अटींचे पालन आवश्यक आहे.
- अधिकृत **Aadhaar Face RD** अॅप फक्त परवानाधारक डिव्हाइस/अॅप्लिकेशनसोबतच काम करते — विनापरवाना
  कोणीही त्याला जोडू शकत नाही.
- अधिक माहिती: https://uidai.gov.in/ecosystem/authentication-devices-documents/ आणि
  UIDAI च्या "Authentication Ecosystem" पानावर अधिकृत प्रक्रिया दिली आहे.

**म्हणजे मी इथे "आधार लिंक करणारा कोड" देऊ शकत नाही** — कारण असा कोड, खरा परवाना नसताना, काम करणारच नाही
आणि चुकीचे आश्वासन ठरेल. जर उद्या तुमची ग्रामपंचायत/संस्था अधिकृतपणे AUA/Sub-AUA म्हणून नोंदणीकृत झाली,
तर एकात्मता खालीलप्रमाणे शक्य होईल (हा फक्त आर्किटेक्चर आहे, कार्यरत कोड नाही):

```
[अॅप] → अधिकृत Aadhaar Face RD अॅप (डिव्हाइसवर वेगळे इन्स्टॉल केलेले)
      → UIDAI च्या Authentication API ला विनंती (तुमच्या AUA परवान्यासह, डिजिटल सही केलेली)
      → UIDAI कडून Yes/No + e-KYC डेटा प्रतिसाद
      → तो प्रतिसाद तुमच्या Apps Script बॅकएंडला पाठवून हजेरी नोंदवणे
```

**सध्याचे face-api.js आधारित चेहरा-मॅचिंग** (वर दिलेले) हे तुमच्या स्वतःच्या डेटावर आधारित,
पूर्णपणे कायदेशीर आणि लगेच वापरता येण्याजोगे आहे — ते आधार e-KYC इतके "सरकारमान्य ओळख" देत नाही,
पण **कर्मचारी तोच आहे का** हे तपासण्यासाठी (जे तुमच्या मूळ गरजेसाठी पुरेसे आहे) चांगले काम करते.

- **HTTPS आवश्यक**: कॅमेरा/GPS हे ब्राउझर फक्त HTTPS वर परवानगी देतो — GitHub Pages आपोआप HTTPS देते, त्यामुळे प्रश्न नाही.

  तर मॉडेल फाईल्स स्वतःच्या repo मध्ये ठेवून `FACE_MODEL_URL` बदला.
- **मोबाईलवर टेस्ट करताना**: Chrome/Safari मध्ये साईट उघडून "Add to Home Screen" केल्यास जवळपास
  नेटिव्ह अॅपसारखा अनुभव मिळतो.

### ❗ "बॅकएंडशी जोडता आले नाही: Api is not defined" ही चूक आली तर

याचा अर्थ `api.js` फाईल ब्राउझरमध्ये लोडच झाली नाही (कोड स्वतः बरोबर आहे — फाईल सापडत नाहीये).
सहसा खालीलपैकी एक कारण असते:

1. **फाईल्स root मध्ये नाहीत** — GitHub repo च्या मुख्य पानावर थेट `index.html`, `api.js`,
   `config.js`, `style.css` वगैरे सर्व एकाच यादीत दिसायला हवेत. जर एखादं आतलं फोल्डर
   (उदा. `hajeri_web_app/`) दिसत असेल आणि त्याच्या आतच या फाईल्स असतील, तर पाथ जुळत नाही.
2. **केस-सेन्सिटिव्हिटी** — GitHub Pages Linux सर्व्हरवर चालते, जिथे `Api.js` आणि `api.js` वेगळ्या
